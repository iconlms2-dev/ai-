1. 시작하기

이 시스템이 뭔가요?

이 시스템은 제품 마케팅에 필요한 콘텐츠를 AI로 자동 생성하고, 배포를 관리하고, 성과를 추적하는 올인원 도구입니다.

블로그 글, 카페 후기, 지식인 답변, 유튜브 댓글, 틱톡 스크립트, 커뮤니티 침투글, 광고 소재까지 — 버튼 몇 번이면 AI가 자동으로 만들어줍니다.

서버 실행 방법

대시보드를 사용하려면 먼저 서버를 켜야 합니다. 방법은 아주 간단합니다:

안티그래비티 폴더를 엽니다

Finder에서 바탕화면 > 안티그래비티 폴더를 엽니다.

마케팅자동화_실행.command 파일을 더블클릭합니다

터미널 창이 열리면서 서버가 자동으로 시작됩니다.

브라우저에서 대시보드를 엽니다

Chrome 등 브라우저에서 http://localhost:8000 주소를 입력합니다.

서버가 돌아가는 동안 터미널 창을 닫지 마세요! 최소화는 괜찮습니다. 작업이 끝나면 터미널을 닫으면 서버가 종료됩니다.

화면 구성

왼쪽 사이드바 — 15개 프로그램 메뉴가 카테고리별로 정리되어 있습니다. 메뉴를 클릭하면 해당 기능이 오른쪽에 나타납니다.

오른쪽 메인 영역 — 선택한 프로그램의 입력/출력 화면입니다. 여기서 키워드를 넣고, 버튼을 누르고, 결과를 확인합니다.

상단 헤더 — 현재 위치(경로)가 표시됩니다.

2. 반드시 알아야 할 개념

프로그램을 사용하기 전에 반드시 이해해야 하는 개념들입니다. 이걸 모르면 버튼은 누를 수 있어도 효과적인 값을 입력할 수 없습니다.

2-1. 구매여정 6단계

고객이 제품을 구매하기까지 거치는 심리 단계입니다. 같은 키워드라도 고객이 어느 단계에 있느냐에 따라 글의 톤이 완전히 달라져야 합니다.

단계이름고객 심리키워드 예시글 톤
0무지문제가 있는지도 모름"전립선이란"부드러운 정보 전달
1인지문제를 인식하기 시작"전립선 증상"공감 + 문제 환기
2호기심해결책을 찾아보기 시작"전립선에 좋은 음식"다양한 방법 비교
3정보습득구체적인 제품을 찾는 중"전립선 영양제"제품 특징 중심
4의심진짜 효과 있는지 의심"전립선 영양제 부작용"근거 제시 + 비교
5구매직전살지 말지 고민 중"전립선 영양제 추천"후기 중심 + 신뢰
키워드 분석 프로그램에서 Gemini AI가 각 키워드의 구매여정 단계를 자동으로 판별해줍니다. 블로그 원고를 생성할 때 이 단계에 따라 글의 오프닝이 자동으로 달라집니다.

2-2. 소구점과 구매원씽

소구점은 "이 제품을 왜 사야 하는지"에 대한 핵심 메시지입니다.

  • 예시: "톱팔메토 320mg 고함량", "식약처 인증 기능성", "3개월 분량 가성비"

구매원씽은 고객이 "아, 이거다!" 하고 결정적으로 마음이 기우는 한 가지 포인트입니다.

  • 예시: "특허받은 원료 조합", "환불 보장", "의사 개발"
제품 정보를 입력할 때 소구점 3~5개와 구매원씽 1개를 명확하게 정리해두면 모든 채널의 콘텐츠 퀄리티가 올라갑니다.

2-3. 나만의 키워드

제품의 브랜드명 또는 고유 키워드입니다. 콘텐츠에 이 키워드를 자연스럽게 1~2회 넣어서 검색 시 우리 제품이 노출되게 합니다.

  • 예시: "프로스테인" (브랜드명), "톱팔메토 골드" (상품명)
나만의 키워드를 너무 많이 넣으면 광고처럼 보입니다. AI가 자동으로 1~2회만 넣도록 설정되어 있지만, 검토 시 확인하세요.

2-4. 금칙어

원고에서 제거해야 할 단어 목록입니다. 원고 생성 시에는 제한 없이 자연스럽게 작성하고, 이후 금칙어 검사로 걸러내는 방식을 사용합니다.

  • 광고 어투: "강추", "대박", "인생템", "꼭 사세요"
  • 법적 위험 표현: "치료", "완치", "예방", "처방" 등 — 맥락상 자연스럽게 나올 수 있지만, 내 제품 홍보에 사용되면 건기식 규정 위반
  • 경쟁사명: 경쟁사 브랜드명
작업 흐름: 원고 생성 (제한 없이) → 금칙어 검사 (셀프모아 2,300개+) → 금칙어 발견 시 AI가 대체어로 수정. 처음부터 막으면 문맥이 부자연스러워질 수 있으므로, "다 뽑고 → 나중에 거르는" 방식입니다.

2-5. 채널 전략과 배정 기준

키워드마다 어떤 채널에 콘텐츠를 만들지 배정합니다. 기본 기준:

채널적합한 키워드목적
블로그검색량 높고 경쟁강도 중~하네이버 블로그 상위노출
카페SEO카페 탭이 상위에 있는 키워드네이버 카페 상위노출
카페바이럴타겟 카페 내부 침투용자연스러운 제품 인지
지식인"~추천", "~좋은거" 등 질문형지식인 상위노출
커뮤니티타겟층이 모여있는 커뮤니티입소문 생성
파워컨텐츠구매 직전 단계(4~5)네이버 파워컨텐츠 광고

2-6. 경쟁강도 / 콘텐츠탭 / 접촉지점

경쟁강도

해당 키워드로 상위노출이 얼마나 어려운지를 나타냅니다.

등급기준의미
상상위글 평균 발행일이 2주(14일) 미만새 글이 매우 빠르게 올라옴. 경쟁 치열.
중2주~4주(28일) 이하적당한 경쟁. 도전 가능.
하4주(28일) 초과오래된 글이 상위. 새 글로 쉽게 밀어낼 수 있음.

콘텐츠탭 순위

네이버 검색 결과에서 블로그/카페/지식인 등 콘텐츠 영역이 몇 번째에 나타나는지입니다. 숫자가 낮을수록 좋습니다(= 위에 나옴).

접촉지점 (= 구매여정 단계)

해당 키워드를 검색하는 사람이 구매여정 6단계 중 어디에 있는지입니다. Gemini AI가 자동 판별합니다.

2-7. 카페바이럴 3단계 침투 전략

같은 카페에 시간차를 두고 3단계로 글을 올려서 자연스럽게 제품 인지를 만드는 전략입니다.

1단계: 일상글
제품 언급 금지
→ 3일 후 →
2단계: 대화침투
카테고리만 언급
→ 3일 후 →
3단계: 제품인지
브랜드 1회 노출
  • 1단계 (일상글): 완전 비상업적. 건강, 피로, 일상 이야기. 계정 신뢰도 쌓기 용도.
  • 2단계 (대화침투): 건강 고민을 공유하면서, 해결책을 카테고리 수준("영양제", "건강기능식품")으로만 언급.
  • 3단계 (제품인지): 나만의 키워드를 1회만 자연스럽게 노출 + 댓글 3~4개로 대화 연출.

2-8. 파워컨텐츠 3단 공식과 BA 기법

파워컨텐츠는 네이버 검색 광고의 한 종류로, 후기글처럼 보이는 광고 랜딩페이지입니다.

멘토 3단 공식

  • 1막 (오프닝): 문제를 격상시키고, 독자가 "맞아, 나도 그래!" 하고 동의하게 만듦
  • 2막 (솔루션): 기존 상식을 깨고, 소거법으로 경쟁 제품을 제거. 우리 제품만 남게 함
  • 3막 (클로징): "지금 안 사면 손해"라는 느낌. FOMO(놓칠까 봐 두려운 마음) 유도

BA(Brilliance Arguments) 설득 기법

결핍 강도를 -1(가벼운 불편)에서 -5(심각한 공포)까지 점진적으로 높여서, 독자가 모르는 사이에 "이걸 안 사면 안 되겠다"는 심리 상태에 이르게 하는 기법입니다.

5. 키워드 분석

🔍
키워드 분석
사이드바 > 분석 > 키워드 분석 · 사용 빈도: 매주 1~2회

목적: 시드 키워드 몇 개를 입력하면, 수백 개로 확장하고, 검색량/경쟁강도/구매여정을 자동 분석합니다.

결과물: 분석이 완료된 키워드 목록이 노션 키워드 DB에 저장됩니다. 이후 블로그/카페/지식인 등에서 이 키워드를 불러와서 콘텐츠를 생성합니다.

Step 1: 키워드 확장 + 검색량 조회

시드 키워드를 입력합니다

입력창에 키워드를 한 줄에 하나씩 입력합니다. 예: "전립선 영양제", "전립선 건강"

"확장 모드"를 선택합니다

라디오 버튼으로 2가지 모드 중 하나를 선택합니다:
• 엑셀 기반 확장 (기본): 엑셀로 업로드한 키워드의 자동완성/연관검색어/함께찾는검색어를 1회만 확장합니다. 빠르고 정확합니다.
• 반복 확장 (BFS): 발견된 키워드를 계속 재확장합니다. 키워드가 부족할 때 사용합니다. 시간이 오래 걸립니다.

"키워드 확장" 버튼을 클릭합니다

선택한 모드에 따라 키워드가 확장됩니다. 실시간으로 발견된 키워드가 화면에 표시됩니다.

검색량이 자동으로 조회됩니다

네이버 검색광고 API를 통해 각 키워드의 PC/모바일 월간 검색량이 조회됩니다.

엑셀 다운로드 → 필터링 → 재업로드

"엑셀 다운로드"로 결과를 받고, 불필요한 키워드를 삭제한 뒤, 다시 "엑셀 업로드"로 올립니다.

검색량이 너무 낮은 키워드(10 미만)는 엑셀에서 필터링해서 제거하세요. 검색량 자체가 없으면 콘텐츠를 만들어도 유입이 발생하지 않습니다.

Step 2: SERP 분석

"분석" 버튼을 클릭합니다

각 키워드마다 네이버 검색 결과를 크롤링하여 분석합니다.

분석 결과 확인

경쟁강도(상/중/하), 콘텐츠탭 순위, 상위글 5개(제목+발행일)가 테이블에 표시됩니다.

Step 3: 구매여정 판별

"접촉지점 판별" 버튼을 클릭합니다

Gemini AI가 각 키워드의 구매여정 단계(0~5)를 자동 판별합니다. 50개씩 배치로 처리됩니다.

결과를 확인하고 필요 시 수정합니다

AI 판별이 맞지 않으면 직접 수정할 수 있습니다.

Step 4: 노션 저장

"노션 저장" 버튼을 클릭합니다

키워드 + 검색량 + 경쟁강도 + 구매여정이 노션 키워드 DB에 저장됩니다.

채널 배정은 키워드 분석에서 자동으로 이루어지지 않습니다. 노션 저장 후 키워드 현황 페이지에서 직접 배정채널을 지정하세요.
키워드 확장은 시간이 많이 걸립니다 (50개 기준 약 10~20분). 중간에 브라우저를 닫지 마세요. 진행 상태가 자동 저장되므로, 중단되어도 이어서 할 수 있습니다.

사이드바 > 콘텐츠 생산 > 카페바이럴 · 사용 빈도: 주 1~2회

목적: 카페 내부에 3단계로 자연스럽게 침투하여 제품 인지를 만듭니다.

결과물: 1세트 = 일상글 + 고민글 + 침투글(+댓글 4개). 보통 3~5세트 생성.

사용 방법

카페 카테고리를 선택합니다

맘카페, 건강카페, 피부카페, 다이어트카페 중 선택하거나 직접 입력합니다.

세트 수를 설정합니다

기본 3세트. 한 카페에 3~5세트면 충분합니다.

"생성 시작" 클릭

각 세트마다 3단계(일상글/고민글/침투글+댓글4개)가 자동 생성됩니다.

결과 검토 후 노션 저장

각 단계별로 별도의 행으로 노션에 저장됩니다.

3단계 침투 예시

카페바이럴은 3단계에 걸쳐 자연스럽게 제품을 인지시킵니다.

단계글 예시핵심
1단계
일상글		"요즘 아버지가 밤에 자꾸 깨셔서 걱정이에요"제품/카테고리 언급 없음. 순수 일상 고민
2단계
고민글		"요즘 피로가 너무 심해서요ㅠㅠ 다들 어떻게 관리하세요?"카테고리 수준만 언급. 제품명 금지. 질문으로 마무리
3단계
침투글+댓글		본문: "친구가 알려준 영양제 먹고 좀 나아졌어요" + 댓글4: "저도 요즘 그거 먹고 있는데 괜찮아요ㅎㅎ"본문에서 제품 1회, 댓글4에서 1회만. 댓글은 공감→경험→질문→정보제공 흐름

작업 모드 선택

직접 작업 또는 외주 작업 모드를 선택할 수 있습니다. 외주 작업 시 실행사에게 전달할 형태로 결과가 정리됩니다. 또한 작업 카페의 URL과 이름을 기록하여 어떤 카페에 침투할지 관리할 수 있습니다.

3단계를 같은 날에 모두 올리면 안 됩니다! 각 단계 사이에 최소 3일 간격을 두세요. 배포 일정 관리에서 자동으로 타이밍을 계산해줍니다.

계정 관리

카페바이럴은 여러 계정으로 운영합니다. 대시보드의 "👤 계정 관리" 탭에서 계정을 등록·삭제하고 상태를 추적합니다.

계정 추가

"+ 추가" 버튼 > 닉네임, 카페 이름, 네이버 ID를 입력합니다.

상태 확인

각 계정의 일상글/고민글/침투글 수, 제품 언급 횟수가 자동 추적됩니다.

3단계 잠금 해제

일상글 5개 + 고민글 1개 이상이어야 3단계(침투글)가 잠금 해제됩니다.

제품 언급 한도

한 계정당 같은 제품 최대 2회까지만 언급할 수 있습니다. 초과 시 자동 차단됩니다.

단계별 개별 생성

"🎯 단계별 생성" 탭에서 계정을 선택한 뒤, 1단계/2단계/3단계를 각각 따로 생성할 수 있습니다. 생성 수도 지정 가능합니다.

2주 플랜

단계별 생성 탭에서 "📅 플랜 생성"을 클릭하면, 2주간의 일정(일상글→댓글교류→고민글→침투글)이 자동 생성됩니다. 일정에 맞춰 단계별로 글을 올리세요.

9. 지식인

❓
지식인
사이드바 > 콘텐츠 생산 > 지식인 · 사용 빈도: 매일

목적: 네이버 지식인에 올릴 자연스러운 질문 + 답변을 생성합니다. 두 가지 모드를 지원합니다.

모드 1: 자문자답

결과물: 질문(제목 25자 이내 + 본문 6~8문장) + 답변 2개(각 10줄 이상, 서로 다른 관점)

키워드 입력 또는 노션에서 로드

"지식인" 또는 "지식인바이럴"로 배정된 키워드가 자동 로드됩니다.

"생성" 클릭

질문 제목 + 질문 본문 + 답변 2개가 한 세트로 생성됩니다.

검토 후 노션 저장

질문에 브랜드명이 없는지, 답변 2개의 말투가 다른지 확인합니다.

모드 2: 직접 답변

결과물: 실제 고민글에 대한 맞춤 답변 1개 (10줄 이상)

"직접 답변" 탭 클릭

지식인 메뉴 상단에서 "💬 직접 답변" 탭을 선택합니다.

고민글 복붙

실제 지식인 고민글 텍스트를 복사해서 입력합니다. 여러 개 추가 가능합니다.

"답변 생성" 클릭

고민글의 구체적인 상황에 맞춘 답변 1개가 생성됩니다.

복사 → 지식인에 답변 등록 → "답변 완료" 체크

생성된 답변을 복사하여 실제 지식인에 답변합니다. 완료 후 체크박스를 눌러 작업을 기록합니다.

직접 답변 핵심 원칙: 답변 시 나만의 키워드만 단독 추천하지 않습니다. 운동, 식단, 생활습관, 다른 성분 등 여러 솔루션을 함께 언급하며 그 중 하나로 자연스럽게 녹여냅니다. 여러 방법을 시도해본 실제 사용자처럼 보여야 합니다.

Q&A 예시

구성예시
질문"전립선 영양제 추천해주세요" (25자 이내, 브랜드명 없음)
답변1"저도 40대 초반에 비슷한 증상 있었는데요, 낙산 성분이 들어간 영양제 먹어보니까 확실히 달라졌어요. 밤에 화장실 가는 횟수가 줄었습니다."
답변2"전립선은 쏘팔메토가 유명한데 최근에는 낙산도 많이 찾더라고요. 저는 활낙산이라는 제품 먹고 있는데 한 달 정도 되니까 좀 편해진 느낌입니다."
답변 2개는 서로 다른 톤(경험형 vs 정보형)으로 생성됩니다. 질문에는 브랜드명이 들어가지 않습니다.
답변 2개는 반드시 다른 관점에서 씁니다. 예: 답변1은 "경험담" 위주, 답변2는 "성분 분석" 위주.

10. 유튜브댓글

▶️
유튜브댓글
사이드바 > 콘텐츠 생산 > 유튜브댓글 · 사용 빈도: 주 2~3회

목적: 타겟 유튜브 영상에 올릴 자연스러운 바이럴 댓글 3단 시나리오를 생성합니다.

결과물: 영상별 댓글 3개 세트 (밑밥 → 해결사 → 쐐기 시나리오)

사용 방법

방법 1: 키워드로 영상 대량 수집 (추천)

키워드 입력

원고생성 탭 상단의 "키워드로 영상 수집" 섹션에서 키워드를 입력하고 수집 수량(기본 50개)을 설정합니다.

"영상 수집" 클릭

yt-dlp를 통해 YouTube에서 키워드 관련 영상을 자동 검색합니다. 제목, URL, 채널명, 조회수가 함께 수집됩니다.

영상 선택

수집된 영상 목록에서 댓글을 달 영상을 체크박스로 선택합니다. "전체 선택"도 가능합니다.

"선택한 영상으로 댓글 생성" 클릭

선택한 영상들의 제목/설명을 일괄 크롤링한 뒤 영상 폼에 자동으로 채워집니다.

"생성 시작" 클릭

각 영상별로 AI가 바이럴 댓글을 생성합니다. 결과를 검토 후 노션에 저장합니다.

방법 2: URL 직접 입력

YouTube URL을 입력합니다

영상 URL을 입력하면 제목, 설명(더보기), 자막을 자동으로 가져옵니다. 크롤링에 실패할 경우 수동 입력도 가능합니다.

"생성" 클릭

영상 내용을 분석한 뒤 3단 시나리오 댓글이 생성됩니다:
① 밑밥 (공감형 질문) — 영상 내용에 공감하며 관련 질문
② 해결사 (키워드 삽입) — 답변 형태로 나만의 키워드 자연스럽게 추천
③ 쐐기 (구매 유도) — "나도 해봐야겠다" 식의 반응 댓글

검토 후 노션 저장

제품명이 자연스럽게 들어갔는지, 해시태그나 링크가 없는지 확인합니다.

3단 시나리오 구조

단계역할예시
댓글1
밑밥		공감형 질문 — 영상 내용에 공감하며 나만의 키워드 관련 질문"저도 이거 고민인데 혹시 좋은 방법 있나요?"
댓글2
해결사		답변형 추천 — 1단계에 대한 답변으로 나만의 키워드 자연스럽게 삽입"나는 이거 써봤는데 확실히 달라지더라"
댓글3
쐐기		구매 행동 유도 — "나도 해봐야겠다" 식 반응"오 진짜? 나도 찾아봐야겠다"
"강추", "인생템" 같은 광고 어투는 자동으로 회피되지만, 혹시 들어가 있으면 수정하세요. 3개 댓글은 반드시 다른 사람이 쓴 것처럼 톤을 달리해야 합니다.

AdsPower 프로필 설정 가이드

1. AdsPower 클라이언트 설치

app.adspower.com에서 Mac용 클라이언트를 다운로드하고 설치합니다.

2. 프로필 생성

"New Profile" 클릭 → Name에 계정 별명 입력 (예: "계정1-lita") → OS는 Mac 선택

3. 프록시 연결

Proxy 섹션에서 Proxy type을 "HTTP"로 선택 → Host:Port에 IPRoyal 프록시 정보 입력 → Username/Password 입력 → "Check Proxy"로 연결 확인

4. 계정 정보 등록

프로필 생성 후 ⋮ → "Edit account" → Platform을 "accounts.google.com" 선택 → Username(이메일), Password, 2FA key(OTP 시크릿) 입력

5. 구글 계정 보안 설정

프로필 Open → accounts.google.com 접속 → 로그인 → 비밀번호 변경, 복구 이메일을 본인 이메일로 변경, 판매자 복구 전화번호 삭제

6. 대시보드 연동

마케팅 자동화 대시보드 > 유튜브댓글 > 자동게시 탭에서 계정 추가 시 AdsPower 프로필 ID를 입력하고, 안티디텍트 모드를 "AdsPower"로 선택

1계정 = 1프로필 = 1프록시: 각 구글 계정은 반드시 고유한 AdsPower 프로필과 프록시에 연결해야 합니다. 같은 프록시로 여러 계정을 사용하면 계정 간 연결이 탐지될 수 있습니다.

필수 패키지 설치

자동게시 기능을 사용하려면 터미널에서 다음 명령어를 실행하세요:

pip3 install playwright playwright-stealth && python3 -m playwright install chromium

Playwright는 자동 게시 시 브라우저를 제어하는 데 사용됩니다. AdsPower 모드에서는 Playwright가 AdsPower 브라우저에 연결만 하므로 별도 Chromium이 필요하지 않습니다.

대시보드에서의 계정 관리

계정 추가

자동게시 탭 > 계정 관리에서 Google 이메일 + 별명 + 프록시(선택) + AdsPower 프로필ID(선택)를 입력합니다.

수동 로그인 (최초 1회)

로그인 테스트

계정 삭제

더 이상 사용하지 않는 계정은 삭제 버튼으로 제거합니다. 쿠키도 함께 삭제됩니다.

🚀 자동게시 (CommentBoost)

목적: 생성한 댓글을 YouTube에 실제로 자동 게시합니다. Playwright + Chromium 브라우저를 사용하여 실제 사용자처럼 댓글을 작성합니다.

자동 게시 실행

Notion의 유튜브 댓글 DB에서 "댓글작업전" 상태인 작업을 자동으로 가져옵니다. 중복 영상은 자동 필터링됩니다.

결과 확인

성공/실패/건너뜀/좋아요 수가 표시됩니다. 작성된 댓글은 Notion에 "댓글완료"로 업데이트되고, 트래킹 목록에 자동 등록됩니다.

IP 관리 (아이폰 테더링)

계정 전환 시 IP를 변경하여 계정 간 연결을 방지합니다. 아이폰 핫스팟 + Wi-Fi 토글 방식을 사용합니다.

동작 방식

대시보드의 "🔄 IP 변경 (테더링)" 버튼 클릭 → macOS Wi-Fi OFF → 3초 대기 → Wi-Fi ON → 아이폰 핫스팟 재연결 → 새 IP 할당

사전 설정

아이폰 핫스팟을 켜고 Mac에서 해당 핫스팟에 연결된 상태여야 합니다. Wi-Fi가 자동으로 아이폰 핫스팟에 재연결됩니다.

IP 변경 없이도 자동게시는 동작합니다. 프록시를 계정별로 설정한 경우 IP 변경이 불필요합니다.

좋아요 구매 (SMM 패널)

댓글 작성 후 SMM 패널(smmwiz)을 통해 좋아요를 구매합니다. 반자동 방식으로 사용자 승인 후 결제됩니다.

설정

자동게시 탭의 "좋아요 자동 구매" 섹션에서 체크박스 ON + 수량을 설정합니다. SMM 잔액도 표시됩니다.

자동 게시 실행

댓글이 성공적으로 게시되면 "좋아요 주문 대기" 목록에 자동 추가됩니다. 이 단계에서는 아직 결제되지 않습니다.

승인

자동 게시 완료 후 "👍 좋아요 주문 승인 대기" 섹션이 표시됩니다. 주문 내역(영상별 수량)을 확인한 뒤 "✅ 전체 주문 승인" 버튼으로 결제합니다.

12. 숏츠 제작 (풀 자동화)

🎬
숏츠 제작
사이드바 > 콘텐츠 생산 > 숏츠 제작 · 10단계 풀 자동화 · 비용: ~$3.5-5.0/영상

목적: 소재 입력부터 YouTube 업로드까지 10단계를 자동화합니다. 사람이 할 일은 CapCut에서 Export 버튼 1번 누르는 것뿐입니다.

10단계 파이프라인

단계내용자동/수동
00. 소재 확인제품/타겟/문제/감정/CTA 검증자동
01. 벤치마킹YouTube URL 크롤링 → 자막/댓글/패턴 추출자동 (URL 선택)

사용 방법 (대시보드)

소재 입력

제품명, 타겟, 핵심문제, 감정, 신뢰근거, CTA를 입력합니다.

Auto: 전 과정 자동. Ask: 전략 선택과 업로드 전에 확인.

Google Drive 연동

TTS 생성 파일(mp3, srt, txt)은 자동으로 Google Drive의 "숏츠_output" 폴더에 업로드됩니다. 업로드 후 로컬 파일은 자동 삭제되어 용량을 절약합니다.

  • 설정: 프로젝트 루트에 credentials.json (Google OAuth) 필요
  • 최초 1회 브라우저 로그인 → 이후 자동 갱신
  • credentials.json 없으면 로컬 저장으로 동작 (기존대로)

08_edit 단계에서 생성된 프로젝트 JSON이 CapCut 로컬 폴더에 자동 등록됩니다.

  • 설정: .env에 CAPCUT_DRAFTS_DIR 경로 필요

월~일 7일이 가로로 나열됩니다. 지난 날짜는 실제 생산량, 오늘은 목표 대비 현황, 미래는 예정 작업이 표시됩니다.

1단계 발행일을 입력하면 2단계(+3일), 3단계(+6일) 예정일이 자동 계산됩니다. 오늘 해야 할 단계가 있으면 빨간색으로 강조됩니다.

채널별 일일 목표 개수를 설정할 수 있습니다. 기본값: 블로그 3, 카페 3, 지식인 5, 바이럴 1세트, 커뮤니티 1.

매출 기반 목표 산출기

월 매출 목표에서 역산하여 필요한 노출 수를 자동 계산합니다. 제품별로 입력 가능합니다.

입력설명
월 매출 목표 (만원)이번 달 달성하고 싶은 매출
객단가 (원)제품 1개 평균 판매가
전환율 (%)유입 대비 구매 비율 (기본 2%)

자동 계산: 매출목표 ÷ 객단가 = 필요주문 → ÷ 전환율 = 필요유입 → ÷ CTR = 필요노출 → ÷ 4 = 주간 필요노출

"+ 제품 추가" 버튼으로 여러 제품을 동시에 관리할 수 있습니다. 입력값은 자동 저장됩니다.

24. 자동 검수

✅
자동 검수 시스템
모든 콘텐츠 생성 시 자동 실행

목적: 콘텐츠 생성 후 자동으로 품질을 검수합니다. 대시보드와 Slack 모두에서 동작합니다.

3단계 검수

  • 1단계: 규칙 검수 (코드) — 글자수, 키워드 횟수, 금지어, 구조 등을 자동 체크. 실패 시 자동 리비전 (최대 3회).
  • 1.5단계: 환각 탐지 — AI가 만들어낸 허위 정보를 탐지합니다. 아래 3계층으로 검사합니다:
    • L1 패턴 탐지: 출처 없는 통계("97% 효과"), 검증 불가 인용("서울대 연구팀"), 가짜 인증("FDA 승인") 등을 정규식으로 자동 탐지
    • L2 제품 대조: 콘텐츠에 언급된 기능/성분이 실제 제품 정보에 있는지 대조. 없는 기능을 썼으면 "제품 불일치" 표시
    • L3 AI 검증: L1/L2에서 발견한 의심 항목을 Gemini가 최종 판단 (기존 AI 검수에 통합, 추가 비용 없음)
  • 2단계: AI 검수 (Gemini) — 자연스러움, 정보성, 광고 비노출 등을 AI가 점수 매김. 기준 미달 시 '초안'으로 분류.

생성 중 진행바 아래에 검수 상태가 실시간으로 표시됩니다:

  • ⚙️ 규칙검수 통과 / ✗ 규칙검수 실패 (실패 항목 표시)
  • ↻ 리비전 1/3 (자동 재생성 중)
  • 🔍 환각 탐지 중... → ⚠️ 환각 의심 2건 발견 (의심 항목 상세 표시)
  • AI 검수 중 (Gemini + 환각 L3 검증)...
  • ✓ 검수 통과 — AI 82점, 환각감점 -6, 최종 76점

실행사 협업 프로세스

노션 콘텐츠 DB를 실행사에게 공유합니다

노션 "공유" 버튼으로 실행사를 초대합니다.

내부: 원고 생성 → 검토 → 생산상태를 "확정"으로 변경

실행사: "확정" 상태 원고를 발행 → 배포상태 "배포완료" + URL 입력

내부: 키워드 현황에서 배포 결과 확인

  1. 제목과 본문을 수정하지 말 것 (키워드 반복수, 글자수가 계산되어 있음)
  2. [이미지N] 위치에 제공된 사진을 순서대로 삽입할 것
  3. [링크] 위치에 제품 URL을 삽입할 것
  4. 발행 후 반드시 노션에 배포 URL과 배포일을 입력할 것
  5. 카페바이럴은 지정된 날짜에 맞춰 발행할 것

22. 문제 해결 가이드

서버가 안 켜져요

이미 실행 중인지 확인

터미널에서 lsof -i :8000 입력. 이미 있으면 lsof -ti:8000 | xargs kill -9로 종료 후 재시작.

다시 실행

마케팅자동화_실행.command 더블클릭. 또는 터미널에서 python3 server.py

그래도 안 되면

에러 메시지를 Claude Code에 복사+붙여넣기. Claude Code가 직접 코드를 수정해줍니다.

대시보드 화면이 안 나와요

  1. 서버가 실행 중인지 터미널에서 확인 ("Uvicorn running" 메시지)
  2. 브라우저 주소가 http://localhost:8000 맞는지 확인
  3. 브라우저 캐시 삭제: Cmd+Shift+R (맥)

키워드 확장이 안 돼요

  1. Chrome이 설치되어 있는지 확인
  2. 터미널에서 pip3 install selenium --upgrade
  3. 네이버가 일시적으로 차단한 경우 → 5분 기다렸다 재시도

노션 저장이 안 돼요

  1. 에러 메시지에 401이 있으면 → 토큰 만료. 노션에서 토큰 재발급 후 CLAUDE.md 업데이트.
  2. 에러 메시지에 404면 → DB ID 오류. CLAUDE.md의 DB ID 확인.
  3. 해당 DB가 인테그레이션에 연결되어 있는지 노션에서 확인.

샤오홍슈 수집이 안 돼요

  1. Chrome에서 샤오홍슈(xiaohongshu.com)에 로그인
  2. 터미널에서: /Users/iconlms/Library/Python/3.11/bin/xhs login
  3. "Cookies extracted from chrome" 메시지 확인
  4. 쿠키는 7일 유효. 7일마다 이 과정을 반복.

AI 생성 퀄리티가 낮아요

  1. 제품 정보의 USP(차별점)가 구체적인지 확인
  2. 소구점과 구매원씽이 막연하지 않은지 확인
  3. 구매여정 단계가 키워드에 맞게 설정되어 있는지 확인
  4. 같은 키워드로 여러 번 생성해서 가장 좋은 것 선택
  5. 카페SEO는 상위글 URL을 직접 넣으면 퀄리티가 올라감

글자수가 안 맞아요

AI가 한글 글자수를 정확하게 못 세는 경우가 있습니다. 서버에서 자동 검증 후 미달 시 3회까지 재생성합니다. 그래도 부족하면 수동으로 보충하세요.

엑셀 다운로드가 안 돼요

  1. 브라우저 팝업 차단 확인: Chrome 설정 → 팝업 및 리디렉션 → localhost 허용
  2. 그래도 안 되면 Claude Code에 "엑셀 다운로드 안 돼"라고 말하세요.

코드 에러가 발생했어요

에러 메시지를 Claude Code에 복붙

Claude Code가 직접 코드를 수정해줍니다.

수정 후 서버 재시작

터미널에서 Ctrl+C → 마케팅자동화_실행.command 더블클릭

안정성 개선 사항 (v2)

다음 안정성 개선이 적용되어 있습니다:

  • 스트리밍 에러 처리: 콘텐츠 생성 중 오류 발생 시, 화면이 멈추지 않고 에러 메시지가 팝업으로 표시됩니다.
  • Claude API 자동 재시도: API 일시 장애(429/500 에러) 시 최대 3회 자동 재시도합니다. 별도 조치 불필요.
  • YouTube 자동게시 안정화: 브라우저 리소스 누수 방지 및 상태 관리 동시성 문제가 수정되었습니다.
  • 파일 저장 안전성: 댓글 추적, 계정 정보 등의 JSON 파일이 원자적으로 저장되어 데이터 손실 위험이 줄었습니다.
  • Toast 알림: 기존 팝업(alert) 방식에서 화면 우측 상단 Toast 알림으로 변경. 알림이 떠도 작업이 멈추지 않습니다.
  • SSE 자동 재연결: 콘텐츠 생성 중 네트워크가 끊기면 자동으로 최대 3회 재연결을 시도합니다.
  • 브라우저 동시 실행 제한: Selenium 브라우저가 동시에 1개만 실행되어 시스템 과부하를 방지합니다.

B. 계정 준비 및 관리

카페바이럴과 커뮤니티 침투는 여러 계정이 필요합니다. 한 계정으로 모든 글을 올리면 광고 계정으로 낙인찍힙니다.

계정 농사 (필수)

새 계정을 바로 마케팅에 사용하면 안 됩니다. 먼저 "계정 농사"를 해야 합니다:

  1. 네이버 계정 생성 (실명 인증 필수)
  2. 타겟 카페에 가입
  3. 1~2주 동안 일상글 5~10개 작성 (출석체크, 공감글 등)
  4. 다른 회원 글에 댓글 5~10개 달기
  5. 그 후에야 카페바이럴 1단계 글 시작
카페바이럴 프로그램의 1단계 "일상글"이 바로 계정 농사용입니다. 시스템이 자동으로 일상글을 생성해줍니다.
동일 IP에서 여러 계정으로 활동하면 패턴이 감지됩니다. VPN이나 모바일 데이터를 번갈아 사용하세요.

C. 다중 제품 운영

제품이 2개 이상일 때:

  1. 노션 제품 DB에 새 제품 행을 추가합니다.
  2. 소구점 DB에 해당 제품의 소구점 3~5개를 등록합니다.
  3. 대시보드 제품 정보에서 새 제품 정보를 입력합니다.
  4. 키워드 분석으로 새 제품의 키워드를 발굴합니다.
  5. 키워드 현황에서 발굴된 키워드에 배정채널을 지정합니다.
제품 정보를 바꾸지 않고 원고를 생성하면, 다른 제품의 정보가 들어간 원고가 나옵니다. 반드시 제품 정보를 먼저 확인하세요!

운영이 안정되면 생산량을 점진적으로 늘립니다:

  1. 키워드 추가: 매주 새 시드 키워드 3~5개씩 확장
  2. 채널 확장: 블로그 → 카페+지식인 추가 → 커뮤니티+틱톡
  3. 제품 추가: 같은 카테고리의 다른 제품 런칭
  4. 실행사 활용: 원고 생성은 내부, 발행은 실행사에 위임
생산량을 급격히 늘리면 네이버가 패턴을 감지합니다. 주당 20~30% 이내로 점진적으로 늘리세요.

E. 저작권 및 법적 주의사항

이미지 저작권

  • 샤오홍슈/바이두에서 수집한 이미지는 저작권 리스크가 있습니다.
  • 얼굴 모자이크는 초상권 보호이지, 저작권 해결이 아닙니다.
  • 블로그/카페 후기글에 참고 이미지로 사용하는 것은 상대적으로 리스크가 낮습니다.
  • 가장 안전한 방법: 자체 촬영 사진 또는 정식 라이선스 이미지 사용.

건기식 광고 규정

주의 표현 (원고에 자연스럽게 나올 수 있지만, 내 제품 홍보 맥락에서는 금칙어 검사로 걸러내야 함):

  • "치료", "완치", "예방", "처방", "투약" — 의약품과 혼동될 수 있음
  • "반드시 효과", "무조건 개선", "부작용 없음" — 100% 보장 표현
  • "의사 추천", "약사가 권하는" — 의료인 추천 표현
이런 표현들은 원고 생성 시 제한하지 않습니다. 맥락상 자연스럽게 나올 수 있는 단어이므로, 다 뽑고 → 금칙어 검사로 거르는 방식을 사용하세요.

허용:

  • 식약처 인증 기능성 문구
  • "저는 이래서 좋았어요" — 개인 경험 표현
  • 성분명과 함량 사실 표기
금칙어를 제품 정보에 입력해두면 AI가 자동으로 피합니다. 하지만 100%는 아니니 검토 시 반드시 확인.

F. 백업 및 복구

백업 방법

가장 간단한 방법: 안티그래비티 폴더를 통째로 복사해서 "안티그래비티_백업_0325" 같은 이름으로 저장.

  • 프로젝트 폴더를 매주 1회 Google Drive 또는 외장하드에 백업
  • 노션 데이터는 노션 클라우드에 자동 저장되므로 별도 백업은 월 1회면 충분
  • 중요한 코드 변경 전에는 반드시 현재 상태를 백업

서버 장애 시 복구

  1. server.py가 안 될 때: 에러 메시지를 Claude Code에 복붙 → 코드 수정 → 서버 재시작
  2. dashboard.html이 깨졌을 때: Cmd+Shift+R로 캐시 삭제. 안 되면 백업에서 복원.
  3. 노션 연결 끊겼을 때: 인테그레이션 활성 확인 → 토큰 재발급 → CLAUDE.md 업데이트
