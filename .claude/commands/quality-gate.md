---

최근 변경사항에 대해 4단계 품질 게이트를 실행합니다:

## 1. 변경 유형 감지
- `git diff --name-only`로 변경된 파일 목록 확인
- 코드 변경: server.py, src/**/*.py 파일이 변경된 경우
- 콘텐츠 변경: job_state.json 또는 콘텐츠 관련 파일이 변경된 경우
- 에이전트 변경: .claude/agents/*.md 파일이 변경된 경우

## 2. 코드 변경 시
- `/verify` 실행 → Stage 1~4 전체 실행 → 코드 품질 점수 산출

## 3. 콘텐츠 변경 시
- 해당 채널의 reviewer 에이전트 실행 → 콘텐츠 품질 점수 산출

## 4. 통합 판정
- 코드 + 콘텐츠 모두 변경 시: 두 점수 중 낮은 쪽이 최종 판정
- 한쪽만 변경 시: 해당 점수가 최종 판정
- **PASS (90+)**: "✅ 품질 게이트 통과 (점수/100)"
- **CONCERNS (70-89)**: "⚠️ 경미한 이슈 있음 (점수/100)" + 감점 내역 + 선택지
- **FAIL (<70)**: "❌ 품질 게이트 미통과 (점수/100)" + 감점 내역 + 수정 필요 항목

## 5. 결과 요약 보고
```
## 품질 게이트 결과: [PASS/CONCERNS/FAIL] (점수/100)

### 코드 (해당 시)
- 점수: N/100
- 감점: ...

### 콘텐츠 (해당 시)
- 점수: N/100
- 감점: ...

### 통합 판정: [PASS/CONCERNS/FAIL]
### 권장 액션: [즉시 진행 / 사용자 확인 필요 / 수정 필요]
```
