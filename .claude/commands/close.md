세션을 마감하고 맥락을 저장합니다. 다음 세션에서 이어서 작업할 수 있도록.

## 절차

### 1. SESSION_LOG.md 확인
- 프로젝트 루트에 `SESSION_LOG.md`가 있으면 append, 없으면 생성
- 기존 항목 수를 세서 세션 번호 부여

### 2. 대화 회고
이번 세션에서 다음을 정리:
- **완료한 작업**: 구체적으로 뭘 했는지 (파일명, 기능명 포함)
- **남은 작업**: 우선순위 순서로 3~5개
- **학습 사항**: 사용자가 수정해준 것, 확인해준 비자명한 선택. "항상 X 해라" / "절대 Y 하지 마라" 형태로. 없으면 "이번 세션 학습 없음"
- **이어가기 지시**: 다음 세션에서 바로 실행할 한 줄 요약

### 3. SESSION_LOG.md에 append
```markdown
## 세션 #N — YYYY-MM-DD

### 완료
- ...

### 남은 작업 (우선순위)
1. ...
2. ...
3. ...

### 학습
- 항상 X 해라 — 이유: ...
- 절대 Y 하지 마라 — 이유: ...

### 이어가기
> 다음 세션 시작 시 이것부터 하세요: ...
```

### 4. 메모리 업데이트
- 학습 사항 중 향후 세션에도 유효한 것은 memory/ 에 피드백으로 저장
- 프로젝트 관련 결정사항은 project 메모리로 저장

### 5. 마감 출력
```
📁 SESSION_LOG.md 저장 완료
📋 남은 작업: (상위 3개)
▶️ 이어가기: (한 줄)
— CLOSE COMPLETE —
```

이후 추가 질문 하지 않고 종료.

## 규칙
- 기존 항목 절대 수정/삭제 금지 (append only)
- 세션 번호는 기존 항목 수 + 1
- 학습 섹션은 비어있어도 반드시 작성 ("이번 세션 학습 없음")
- /close 호출 시에만 실행. 자동 실행 금지
