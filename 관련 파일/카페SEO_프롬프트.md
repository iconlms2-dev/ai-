# 카페SEO 원고 프로그램 구현 지시서

이 파일을 읽고 dashboard.html의 "카페SEO 원고" 메뉴에 기능을 구현해줘.
CLAUDE.md 파일에 노션 API 토큰, DB ID 등 설정값이 있으니 참조해.

---

## 1. UI 구성

"카페SEO 원고" 메뉴 클릭하면 나오는 화면:

### 입력 영역
1. **키워드 입력 모드 2개 탭**: "키워드 직접 입력" / "노션 DB에서 가져오기"
   - 직접 입력: 텍스트 영역 (한 줄에 키워드 하나)
   - 노션에서 가져오기: "가져오기" 버튼 → 키워드 DB에서 배정채널이 "카페SEO"이고 상태가 "미사용"인 키워드 목록을 불러옴 → 체크박스로 선택

2. **경쟁사 카페글 URL 입력** (선택사항):
   - 텍스트 영역 (한 줄에 URL 하나)
   - URL 있으면 → 원글 제목+본문 크롤링해서 리라이팅
   - URL 없으면 → 새로 생성

3. **제품 정보 영역** (블로그 원고와 동일, localStorage에서 자동 로드):
   - 제품명
   - 주요 성분
   - 핵심 특징 (USP)
   - 타겟층
   - 제품 URL (CTA 링크용)
   - "나만의 키워드" (예: 순호박즙)
   - 금칙어 (쉼표 구분)
   - 서브 키워드 (쉼표 구분)

4. **추가 설정**:
   - 메인키워드 반복횟수 (기본값: 5)
   - 사진 수 (기본값: 8)

5. **"생성 시작" 버튼** + 프로그레스바

### 결과 영역
- 생성된 글 목록. 각 글마다:
  - 제목 (리라이팅 결과)
  - 본문 (펼치기/접기)
  - 댓글 10개 (펼치기/접기)
  - "노션 저장" 버튼 / "제목 복사" / "본문 복사" / "댓글 복사" 버튼
- 하단에 "전체 노션 저장" 버튼

## 2. 백엔드 로직

키워드 하나당 3단계로 처리:

### STEP 1: 제목 리라이팅 (멘토 89번 로직)

경쟁사 카페글 URL이 있으면 원글 제목을 크롤링해서 리라이팅.
URL이 없으면 키워드 기반으로 새 제목 생성.

AI에게 보내는 프롬프트:
```
역할: 네이버 카페 상위노출을 위한 제목 리라이팅 전문가.

목표: 경쟁사 카페글 제목을 참고하여, 메인 키워드를 유지하면서 클릭을 유도하는 리라이팅 제목 1개를 생성한다.

입력:
- 기존 제목: {원글 제목 또는 빈칸}
- 메인 키워드: {키워드}

리라이팅 규칙:
1. 제목의 전체 구조는 유지
2. 마지막 말(어미 or 단어)만 자연스럽게 바꿀 것
3. 클릭을 유도할 수 있는 궁금형 or 공감형 문장으로 변환
4. 메인 키워드는 제목에 그대로 유지
5. 광고 느낌 금지: "추천", "인생템", "꼭 사세요" 등 제외
6. 이모티콘 사용 금지

출력: 리라이팅된 제목 1개만 출력 (부연 설명 없이)

금지사항:
- 제목 외 부가 텍스트 출력 금지
- 이모티콘 금지
- 광고성 표현 금지
```

### STEP 2: 본문 생성 (멘토 88번 로직)

AI에게 보내는 프롬프트:
```
역할: 네이버 카페 상위노출 기준에 맞춰, 결핍 고민과 증상 중심의 질문형 후기 본문을 작성하는 카피라이터.

목표: 광고 티 없이 누구나 공감할 수 있는 카페 질문형 후기 글을 작성한다.

입력:
- 제목: {STEP 1에서 만든 제목}
- 메인 키워드: {키워드}
- 참고 경쟁사글: {원글 본문 또는 빈칸. 빈칸이면 새로 생성}
- 메인 키워드 반복횟수: {설정값}
- 서브 키워드: {입력값}
- 금칙어 목록: {입력값}
- 사진 수: {설정값}

글자수 조건:
- 공백 제외 1,500자 ±100자
- 줄바꿈은 3~5줄 간격 유지

키워드 최적화:
- 메인 키워드: {반복횟수} + 1회 본문 삽입
- 서브 키워드: 각 1회 이상 삽입
- 키워드 누락 시 실패 처리
- 금칙어 포함 시 실패 처리

본문 구성:
1. 오프닝 — 증상과 문제 얘기, 공감 유도
2. 기존 해결 실패 경험
3. 새로운 제품/방법 선택과 고민
4. 질문형으로 마무리

사진 삽입 규칙:
- 문단과 문단 사이에 [어울릴 사진: {짧은 설명}] 형태로 삽입
- 총 {사진수} + 1개 삽입

톤앤매너:
- 자연스러운 후기·질문 톤
- 진솔한 체험 공유 느낌
- 광고처럼 보이면 안 됨

출력: 제목 없이 바로 본문만 출력

금지사항:
- "강추", "인생템", "꼭 사세요" 등 광고 어투 금지
- 금칙어 목록에 포함된 표현 사용 금지
- 키워드 누락 금지
- 출력 시 제목 포함 금지
```

### STEP 3: 댓글 10개 생성 (멘토 92번 로직)

AI에게 보내는 프롬프트:
```
역할: 네이버 카페 글에 달릴 자연스럽고 신뢰도 있는 댓글 10개를 생성하는 작가.

목표: 실제 사용자의 다양한 반응처럼 보이는 댓글 10개를 생성한다.
공감 → 차선책 언급 → 해결책 제시 → 제품 경험담 → 추가 공감·마무리 흐름을 만든다.

입력:
- 실제 본문: {STEP 2에서 만든 본문}
- 메인 키워드: {키워드}
- 제품명 or 간접 언급: {나만의 키워드}

댓글 흐름 설계 (10개):

[1~2번 댓글: 공감 + 차선책 언급]
- 질문자 고민에 깊은 공감 + 흔히 시도하는 차선책(식이요법, 영양제, 한약, 시술, 운동 등) 언급
- 본인도 열심히 해봤지만 효과 제한적이었다는 식으로 작성
- 이 단계에서는 특정 제품 언급이나 광고 티가 나면 안 됨

차선책 목록 (참고용):
· 자연 식이요법 — 한계: "음식만으로는 한계가 있더라."
· 기본 영양제/비타민 — 한계: "일반 영양제만으론 부족하더라구요."
· 전통 방법(한약, 디톡스) — 한계: "민간요법 해봤지만 큰 효과 없었어요."
· 피부과 시술/클리닉 — 한계: "시술도 잠깐일 뿐 근본적으로 안 달라지더라구요."
· 운동/식단 — 한계: "실천이 거의 불가능하더라구요."
· 다이어트 보조제 — 한계: "부작용 걱정되서 시도도 못 하겠더라구요."

[3~4번 댓글: 해결책 전환 (제품명 없이)]
- "방법이 없진 않더라"는 뉘앙스로 희망 제시
- 건강기능식품(특허 성분, 기능성 입증 등) 언급하되, 제품명은 아직 직접 언급하지 않음
- 정보를 알려주는 느낌: "요즘 영양제가 잘 나오더라구요. 비타민제랑 다르게 특허 성분으로..."
- 키워드: "특허 원료", "기능성 입증", "건강기능식품" 등 포함

[5~7번 댓글: 제품명 등장 + 경험담]
- {나만의 키워드}를 자연스럽게 등장시킴
- 실제 사용 경험담 공유: 구체적 변화
- 각각 다른 사람의 입장에서 작성
- 링크, 구매 강조 금지

[8~10번 댓글: 추가 공감 + 마무리]
- 꾸준한 자기관리 중요성 강조
- "방치하면 더 악화된다"는 경고를 살짝 포함
- "많은 사람들이 효과 봤다"는 사회적 증거로 긍정적 마무리
- 독자가 "나도 해봐야겠다"는 마음이 들도록 유도

작성 조건:
- 각 댓글은 서로 다른 사람이 쓴 것처럼 자연스럽게 표현
- 동일한 패턴 반복 금지
- 각 댓글에 메인 키워드 1회 이상 포함
- 이모티콘(ㅎㅎ, ㅠㅠ, ;;, 👍 등) 적절히 활용
- 일상적인 존댓말 체, 편안하고 친근한 어조

출력 형식:
- 한 줄에 한 개의 댓글
- 댓글 사이 줄바꿈은 1회만
- 총 10개 댓글

금지사항:
- "강추", "무조건", "최고" 등 과장 표현 금지
- 구매 유도 문구 금지
- 링크 삽입 금지
```

## 3. 노션 저장

"노션 저장" 버튼 클릭 시 콘텐츠 DB에 저장.

저장 필드:
- 제목: 생성된 제목
- 채널: "카페"
- 본문 요약: 본문 앞 300자
- 댓글_본문: 댓글 10개 전체 텍스트
- 생산 상태: "초안"
- 배포 상태: "미배포"
- 키워드: 키워드 DB 릴레이션 연결
- 사진수: 설정값
- 메인키워드_반복회수: 설정값
- 서브_키워드: 입력값
- 금칙어_목록: 입력값
- 원글_제목: 경쟁사 원글 제목 (URL 입력한 경우)
- 원글_본문: 경쟁사 원글 본문 (URL 입력한 경우)

## 4. 기술 요구사항
- 기존 dashboard.html + server.py를 수정
- 경쟁사 카페글 크롤링은 requests + BeautifulSoup
- 콘텐츠 생성은 AI API 호출 (제목 → 본문 → 댓글 순서대로 3번 호출)
- 각 STEP의 결과가 다음 STEP의 입력으로 자동 전달
- 제품 정보는 localStorage에 저장 (블로그 원고와 공유)
- 키워드 여러 개 넣었을 때 하나 완료되면 바로 결과에 추가
