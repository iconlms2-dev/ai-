# 카페바이럴 프로그램 구현 지시서

이 파일을 읽고 dashboard.html의 "카페바이럴" 메뉴에 기능을 구현해줘.
CLAUDE.md 파일에 노션 API 토큰, DB ID 등 설정값이 있으니 참조해.

카페SEO와 다른 점: SEO는 검색 상위노출용이고, 바이럴은 카페 내부에서 자연스럽게 침투하는 3단계 구조.

---

## 1. UI 구성

"카페바이럴" 메뉴 클릭하면 나오는 화면:

### 입력 영역
1. **카페 카테고리 선택**: 드롭다운 (맘카페 / 건강카페 / 피부카페 / 다이어트카페 / 직접입력)

2. **제품 정보 영역** (localStorage에서 자동 로드, 블로그/카페SEO와 공유):
   - 제품명
   - 주요 성분
   - 핵심 특징 (USP)
   - 타겟층
   - "나만의 키워드" (예: 순호박즙)
   - 타겟 고민 (예: "잦은 소변", "전립선 불편감")

3. **생성 옵션**:
   - 생성할 세트 수 (기본값: 3세트)
   - 3단계 전부 생성 / 단계별 개별 생성 선택

4. **"생성 시작" 버튼** + 프로그레스바

### 결과 영역
세트별로 카드 형태로 표시:

```
📦 세트 1
├── [1단계] 일상글: 제목 + 본문 (펼치기/접기)
├── [2단계] 대화침투글: 제목 + 본문 (펼치기/접기)
└── [3단계] 제품인지글: 제목 + 본문 + 댓글 3~4개 (펼치기/접기)
```

각 단계마다 "복사" 버튼. 세트 전체 "노션 저장" 버튼.

---

## 2. 백엔드 로직 — 3단계 침투 세트

세트 하나당 3단계 글을 순서대로 생성:

### 1단계: 일상글 (계정 신뢰 구축용)

AI에게 보내는 프롬프트:
```
역할: 네이버 카페 내부 회원처럼 보이는 일상글을 작성하는 작가.

목표: 카페 계정의 글 이력을 자연스럽게 채우기 위한 비상업적 일상글을 생성한다. 제품·브랜드 언급 없이, 평범한 카페 회원의 일상 공유글처럼 보여야 한다.

입력:
- 카페 카테고리: {선택값}
- 글 유형: 일상, 질문, 출석체크, 공감, 정보공유 중 랜덤 선택

작성 규칙:
1. 건강, 육아, 피로, 일상 등 비상업적 주제만 다룰 것
2. 2~5문장 이내의 짧은 글
3. 감정 표현(ㅎㅎ, ㅠㅠ, ;;) 자연스럽게 사용
4. 카페 분위기에 맞는 존댓말 체
5. 제품명, 브랜드명, 카테고리명도 언급 금지

출력: 제목 1개 + 본문 1개

출력 예시:
- 제목: "오늘 너무 피곤하네요ㅠ"
- 본문: "애 재우고 나면 체력이 바닥이에요… 다들 이럴 때 어떻게 하세요?ㅠㅠ"

금지사항:
- 제품·브랜드·카테고리 언급 일체 금지
- 광고성 문체 금지
- 링크·해시태그 금지
```

### 2단계: 대화침투글 (자연스러운 대화 침투)

AI에게 보내는 프롬프트:
```
역할: 네이버 카페에서 생활 속 고민을 공유하는 회원처럼 보이는 글을 작성하는 작가.

목표: 광고 티 없이 "생활 속 고민"에 초점을 맞춘 공감형 글을 생성한다. 제품명은 아직 금지. 카테고리 수준까지만 허용.

입력:
- 카페 카테고리: {선택값}
- 타겟 고민: {입력값}

작성 규칙:
1. 감정형 or 공감형 제목
2. 본문 구조:
   - 오프닝: 일상적 상황 묘사
   - 중반: 감정 공유 + 기존에 시도한 방법
   - 후반: 간접적 해결 언급 (카테고리 수준만. 예: "천연 성분 영양제", "두피관리제")
   - 마무리: 질문 or 공감 유도
3. 5줄 이하 유지
4. 이모티콘(ㅎㅎ, ㅠㅠ, 👍) 1~2개 자연스럽게 사용
5. 존댓말, 따뜻한 대화체

출력: 제목 1개 + 본문 1개

금지사항:
- 제품명·브랜드명 언급 금지 (카테고리 수준까지만 허용)
- 링크·해시태그 금지
- 광고 문체 금지
- "추천드려요", "이거 진짜 좋아요", "꼭 써보세요" 등 광고 어투 금지
```

### 3단계: 제품인지글 + 댓글 세트 (제품 인지 스며듦)

AI에게 보내는 프롬프트:
```
역할: 네이버 카페에서 경험을 공유하는 회원처럼 보이는 글과, 그 글에 달릴 자연스러운 댓글 세트를 작성하는 작가.

목표: 제품명(나만의 키워드)을 노출할 수 있지만, 홍보 톤이 아닌 경험 공유형으로 작성한다. 글 1개 + 댓글 3~4개를 세트로 생성하여 자연스러운 대화 흐름을 만든다.

입력:
- 카페 카테고리: {선택값}
- 타겟 고민: {입력값}
- 나만의 키워드: {입력값}
- 제품명: {입력값}
- 핵심 특징: {입력값}

글 작성 규칙:
1. 제목: 감정형 or 공감형
2. 본문 구조:
   - 오프닝: 고민 상황 묘사
   - 중반: 기존 시도 + 실패 경험
   - 후반: 나만의 키워드를 경험 공유 형태로 1회만 자연스럽게 언급
   - 마무리: 질문으로 댓글 유도
3. 5줄 이하 유지
4. 이모티콘 자연스럽게 사용

댓글 작성 규칙:
1. 3~4개 댓글 생성
2. 각 댓글은 서로 다른 사람이 쓴 것처럼 보여야 함
3. 댓글 흐름: 공감 → 경험공유 → 자연스러운 연결
4. 나만의 키워드는 댓글 후반부에 1회만 노출
5. 각 댓글은 1~2문장
6. 이모티콘(ㅎㅎ, ㅠㅠ, 👍) 적절히 사용

출력 형식:
[글]
제목: ...
본문: ...

[댓글]
1. ...
2. ...
3. ...
4. ...

톤앤매너:
- 존댓말, 따뜻한 대화체 ("~했어요ㅎㅎ", "~하더라구요ㅠ")
- 문장 길이: 2~3문장 단락 / 5줄 이하 유지
- 대화 흐름: 질문 → 공감 → 경험공유 → 마무리

금지사항:
- 광고 어투 금지: "추천드려요", "이거 진짜 좋아요", "꼭 써보세요"
- 직접 링크 삽입 금지
- 해시태그·쇼핑몰 언급 금지
- 동일 문체 반복 금지 (자동 광고판정 위험)
- 제품명 과다 노출 금지 (글+댓글 전체에서 1~2회만)
- 계정간 대화 반복 흔적 금지 (같은 사람이 쓴 티 나면 제재)
```

---

## 3. 노션 저장

"노션 저장" 버튼 클릭 시 콘텐츠 DB에 세트 단위로 저장.

1단계, 2단계, 3단계 각각 별도 행으로 저장:

공통 저장 필드:
- 채널: "카페"
- 프로젝트: "P8"
- 생산 상태: "초안"
- 배포 상태: "미배포"

1단계:
- 제목: 일상글 제목
- 본문 요약: 일상글 본문

2단계:
- 제목: 대화침투글 제목
- 본문 요약: 대화침투글 본문

3단계:
- 제목: 제품인지글 제목
- 본문 요약: 제품인지글 본문
- 댓글_본문: 댓글 3~4개 전체 텍스트

---

## 4. 기술 요구사항
- 기존 dashboard.html + server.py를 수정
- 콘텐츠 생성은 AI API 호출 (1단계 → 2단계 → 3단계 순서대로 3번 호출)
- 세트 수만큼 반복 (3세트면 9번 AI 호출)
- 각 세트 완료되면 바로 결과에 추가
- 제품 정보는 localStorage에서 자동 로드 (블로그/카페SEO와 공유)
- 1단계~3단계는 같은 카페에 시간차를 두고 올리는 용도이므로, 3단계 글 사이에 톤/문체가 달라야 함
