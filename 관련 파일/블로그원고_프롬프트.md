# 블로그 원고 프로그램 구현 지시서

이 파일을 읽고 dashboard.html의 "블로그 원고" 메뉴에 기능을 구현해줘.
CLAUDE.md 파일에 노션 API 토큰, DB ID 등 설정값이 있으니 참조해.

---

## 1. UI 구성

"블로그 원고" 메뉴 클릭하면 나오는 화면:

### 입력 영역
1. **키워드 입력 모드 2개 탭**: "키워드 직접 입력" / "노션 DB에서 가져오기"
   - 직접 입력: 텍스트 영역 (한 줄에 키워드 하나)
   - 노션에서 가져오기: "가져오기" 버튼 → 키워드 DB에서 배정채널이 "블로그SEO"이고 상태가 "미사용"인 키워드 목록을 불러옴 → 체크박스로 선택

2. **구매여정 단계 선택**: 드롭다운 (0_무지 / 1_인지 / 2_호기심 / 3_정보습득 / 4_의심 / 5_구매직전)
   - 노션에서 가져온 키워드는 DB에 저장된 구매여정_단계 값을 자동으로 적용

3. **제품 정보 영역** (한번 입력하면 저장돼서 다음에 자동 로드):
   - 제품명
   - 주요 성분
   - 핵심 특징 (USP)
   - 타겟층
   - 제품 URL (CTA 링크용)
   - "나만의 키워드" (예: 순호박즙)
   - 금칙어 (쉼표 구분)

4. **"생성 시작" 버튼** + 프로그레스바

### 결과 영역
- 생성된 글 목록. 각 글마다:
  - 제목
  - 본문 (펼치기/접기)
  - 사용된 키워드, 구매여정 단계, 사진수, 키워드반복수 표시
  - "노션 저장" 버튼 / "복사" 버튼
- 하단에 "전체 노션 저장" 버튼

---

## 2. 백엔드 로직

키워드 하나당 아래 흐름으로 처리:

### STEP 1: 상위글 분석 (자동)
키워드로 네이버 PC 통합검색 실행. 신뢰도통합/VIEW 영역의 상위 1~3등 글에서:
- 평균 사진 수 → photo_count (이미지 태그 개수로 추정)
- 평균 키워드 반복 횟수 → keyword_repeat (본문에서 키워드 출현 횟수)
- 상위글 포맷(블로그/카페/지식인) 확인

분석이 어려우면 기본값 사용: photo_count=8, keyword_repeat=5

### STEP 2: 제목 생성
AI에게 아래 규칙으로 제목 1개를 생성시킨다:

```
제목 생성 규칙:
- 키워드를 제목에 반드시 1회 자연스럽게 포함
- 후기처럼 보이게 작성 (예: "진짜 이틀 만에 해결됨", "저만 그런 줄 알았어요")
- 기호, 과장 표현 금지: "대박템", "강추" 등 제외
- 말투는 자연스럽고 말하듯이, 궁금증이나 결과 중심
- 한 문장으로 간결하게
- 이모티콘 금지
```

### STEP 3: 본문 생성
AI에게 아래 규칙으로 본문을 생성시킨다:

#### 글자수 조건
- 전체 분량: 공백 포함 2,200자 이상
- 단락 수: 최소 8단락 이상
- 각 단락: 200~350자 내외
- 절대 압축 요약하지 말 것. 느낌, 사례, 생각, 비교 등을 풍부하게 풀어 쓸 것.

#### 오프닝 — 구매여정 단계별 분기 (★ 핵심)
입력된 구매여정_단계 값을 보고, 해당하는 심리로 오프닝을 작성:

**0_무지 / 1_인지 (문제 인식):**
"증상이 자꾸 생기는데 왜 그런지 몰랐어요."
→ 공감 중심, 불편한 일상 묘사

**2_호기심 (해결 탐색):**
"이것저것 알아보다가 여러 방법을 찾아봤어요."
→ 다양한 방법 탐색 + 문제 간략 언급

**3_정보습득 (해결방법 선택):**
"여러 가지 다 실패하다가 결국 이 제품을 알게 됐어요."
→ 제품 선택 계기 + 기대감 묘사

**4_의심 (비교 검토):**
"경쟁사 제품에는 ○○이 없어서 아쉬웠는데, 이 제품은 ○○이 있어서 만족했어요."
→ 경쟁 제품 언급 + 자사 USP 강조

**5_구매직전 (구매 직전):**
"후기들을 보다가 제 기준엔 이게 괜찮더라구요."
→ 사용 후기 중심 + 신뢰 강조 + 링크 클릭 심리 유도

#### 본문 전개
- 경쟁 제품의 단점 서술
- 내가 선택한 제품 설명
- USP는 리뷰처럼 자연스럽게 녹이기 (예: "성분이 순해서 그런지 덜했어요.")
- "나만의 키워드"가 입력돼있으면 본문에 자연스럽게 포함

#### 이미지 삽입 위치
- 본문 중간에 [이미지1], [이미지2]... 형태로 삽입 위치 표시
- 총 photo_count 값만큼 삽입

#### 키워드 사용 규칙
- 키워드를 전체 글에서 keyword_repeat 값만큼만 삽입
- 억지 삽입 없이 문맥 자연스럽게 유지

#### 클로징
- 후기처럼 마무리
- 직접적 구매 유도 문장 금지, 제안형 문장 사용
- 마지막에 제품 URL 링크 삽입
- 아래 CTA 문장 중 무작위 1개만 선택하여 출력:
  1. "이번주까지만 특가 이벤트 진행한다고 해요. 링크 남겨드릴테니 한번 보시는것도 좋을것 같네요"
  2. "이제품 첼린지도 하고 난리났더라구요. 링크 남겨드릴테니 후기랑 상세페이지 잘 비교해보시고 구매하시길 바라요"
  3. "이번에 못 사면 다음 입고까지 한참 기다려야 한다고 하네요ㅠㅠ 제가 구매했던 링크 남겨드릴게요"
  4. "저는 30%할인할때 샀는데 지금도 하는지는 모르겠네요. 제가 구매했던 링크 남겨드릴게요"
  5. "단순 건강식품이 아니라 기능성 인정 받은 제품이라 믿고 구매했습니다."

#### 톤앤매너
- 존댓말
- 광고성 과장 배제
- 정보 + 체험 혼합 톤 (진솔한 후기)

#### 금지사항
- 제목을 본문에 다시 출력하지 말 것
- "강추", "대박", "인생템" 등 광고 어투 금지
- CTA 문장 2개 이상 출력 금지
- 금칙어 입력창에 입력된 표현 사용 금지

### STEP 4: 노션 저장
"노션 저장" 버튼 클릭 시 콘텐츠 DB에 저장.

저장 필드:
- 제목: 생성된 제목
- 채널: "블로그"
- 본문 요약: 본문 앞 300자
- 생산 상태: "초안"
- 배포 상태: "미배포"
- 키워드: 키워드 DB 릴레이션 연결
- 사진수: photo_count 값
- 키워드_반복수: keyword_repeat 값

---

## 3. AI 원고 생성 방법

콘텐츠 생성은 현재 안티그래비티에서 사용 중인 AI 모델(Claude)을 호출해서 처리.
위의 STEP 2, 3 규칙 전체를 시스템 프롬프트로 넣고, 키워드 + 제품정보 + 구매여정단계 + photo_count + keyword_repeat을 입력값으로 전달.

---

## 4. 기술 요구사항
- 기존 dashboard.html + server.py를 수정해서 블로그 원고 메뉴에 기능 추가
- 상위글 분석은 requests + BeautifulSoup (기존 키워드 분석과 동일 방식)
- 콘텐츠 생성은 AI API 호출
- 노션 저장은 Notion API (CLAUDE.md에 토큰/DB ID 있음)
- 키워드 여러 개 넣었을 때 하나 생성 끝나면 바로 결과에 추가 (전부 끝날 때까지 안 기다림)
- 제품 정보는 localStorage에 저장해서 다음에 자동 로드
