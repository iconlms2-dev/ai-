## 1단계: 기본 설정 (30분)

### 1-1. Notion 콘텐츠 DB 필드 확인/추가

1. Notion 열기
2. 콘텐츠 DB 페이지로 이동
3. 테이블 맨 오른쪽 "+" 클릭해서 현재 속성 목록 확인
4. 아래 필드들이 있는지 체크

| 필드명 | 타입 | 없으면 이렇게 추가 |
|--------|------|-------------------|
| 제목 | Title | 기본으로 있음 |
| 채널 | Select | 옵션: 블로그, 카페, 카페SEO, 지식인, 유튜브, 틱톡, **쓰레드**, 커뮤니티, 파워컨텐츠, 메타광고 |
| 생산 상태 | Select | 옵션: 초안, 검수중, 수정필요, 완성 |
| 배포 상태 | Select | 옵션: 미배포, 배포완료, 발행완료 |
| 키워드 | Relation | 키워드 DB와 연결 |
| 본문 요약 | Text (Rich text) | |
| 배포URL | URL | |
| 배포일 | Date | |
| 생산일 | Date | |
| **작업계정** | **Select** | **새로 추가 필요** |

"작업계정" 필드 추가 방법:
- 테이블 맨 오른쪽 "+" 클릭
- "New property" 클릭
- 이름: `작업계정`
- 타입: Select
- 옵션은 비워두고 저장 (나중에 계정 이름이 자동 추가됨)

### 1-2. Notion 키워드 DB 필드 확인

1. 키워드 DB 페이지로 이동
2. "배정 채널" 속성 클릭
3. Multi-select 옵션 목록에 아래가 전부 있는지 확인:
   - 블로그
   - 카페SEO (또는 카페)
   - 지식인
   - **쓰레드** ← 없으면 추가

추가 방법: 속성 편집 → 옵션 추가 → "쓰레드" 입력 → 저장

### 1-3. 패키지 설치

터미널 열고:
```bash
cd ~/Desktop/안티그래비티
pip3 install -r requirements.txt
playwright install chromium
```

설치 확인:
```bash
python3 -c "import fastapi; import playwright; print('OK')"
```
`OK` 나오면 완료.

## 2단계: Threads 연동 (1~2시간)

### 2-1. Meta Developer 계정 만들기

1. https://developers.facebook.com/ 접속
2. 오른쪽 상단 "로그인" → Facebook 계정으로 로그인
   - Facebook 계정이 없으면 먼저 만들기
3. 로그인 후 "시작하기" 또는 "My Apps" 클릭
4. 개발자 등록 안 했으면 등록 절차 진행
   - 이름 확인
   - 이메일 확인
   - 약관 동의

### 2-2. 앱 만들기

1. https://developers.facebook.com/apps/ 접속
2. "앱 만들기" 버튼 클릭
3. 사용 사례 선택: "기타" 또는 "비즈니스"
4. 앱 유형: "비즈니스" 선택
5. 앱 이름: "안티그래비티 쓰레드" (아무거나)
6. 연락처 이메일: 본인 이메일
7. "앱 만들기" 클릭

### 2-3. Threads API 추가 + 권한 설정

1. 앱 대시보드가 열림
2. 왼쪽 메뉴에서 "제품 추가" 또는 화면에 "제품" 섹션
3. "Threads" 찾아서 "설정" 클릭
4. "Threads 권한" 섹션에서 아래 4개 활성화:
   - ☑ threads_basic
   - ☑ threads_content_publish
   - ☑ threads_manage_insights
   - ☑ threads_manage_replies
5. 저장

### 2-4. 리디렉션 URI 설정

1. 왼쪽 메뉴: Threads API → 설정
2. "유효한 OAuth 리디렉션 URI" 섹션
3. 아래 URL 입력:
   ```
   http://localhost:8000/api/threads/callback
   ```
4. 저장

### 2-5. App ID / Secret → .env 입력

1. 왼쪽 메뉴: 설정 → 기본 설정
2. "앱 ID" 복사
3. "앱 시크릿 코드" → "표시" 클릭 → 복사

터미널에서:
```bash
cd ~/Desktop/안티그래비티
open .env
```

`.env` 파일에서 아래 두 줄 찾아서 값 입력:
```
THREADS_APP_ID=여기에_앱_ID_붙여넣기
THREADS_APP_SECRET=여기에_앱_시크릿_붙여넣기
```
저장 후 닫기.

### 2-6. 테스트 계정 추가

1. Meta 개발자 대시보드로 돌아감
2. 왼쪽: Threads API → 설정
3. "Threads 테스터" 섹션
4. "테스터 추가" → 본인 Threads 계정 (@username) 입력
5. 해당 Threads 계정에서 초대 수락
   - Threads 앱 → 설정 → 보안 → 앱 및 웹사이트 → 초대 수락

> **참고: 앱 심사**
> - 개발 모드에서는 본인 + 테스터 계정만 사용 가능
> - 동조 계정 4~5개 전부 연결하려면 앱 심사 필요 (1~2주 소요)
> - 우선 본인 계정 1개로 테스트 → 심사 통과 후 나머지 연결

### 2-7. OAuth 연결 테스트

1. 서버 시작:
   ```bash
   cd ~/Desktop/안티그래비티
   python3 server.py
   ```
2. 브라우저: http://localhost:8000
3. 사이드바: 🧵 쓰레드
4. "계정 관리" 탭
5. "+ 계정 추가" 클릭
   - 이름: 본인 이름
   - 나이: 입력
   - 직업: 입력
   - 말투: 친근
   - 관심사: 쉼표 구분
   - 참조계정: @참조계정 (쉼표 구분)
   - 메인 계정? → 확인
   - 하루 게시 수: 3
6. 계정 카드에서 "OAuth 연결" 클릭
7. Threads 로그인 → 권한 허용 → 자동 복귀
8. "● 연결됨" 표시되면 성공

**안 되면 체크:**
- .env에 App ID / Secret 정확한지
- 리디렉션 URI가 `http://localhost:8000/api/threads/callback` 인지
- 테스트 계정 초대를 Threads에서 수락했는지

## 4단계: 제품 정보 입력 (10분)

1. 대시보드 아무 생성 탭 열기 (블로그, 일괄 생성 등)
2. "제품 정보" 섹션에 입력:

```
제품명: 예) 닥터스 유산균
나만의 키워드: 예) 닥터스프로바이오
핵심 특징(USP): 예) 100억 CFU 보장, 장까지 도달하는 코팅 기술
타겟층: 예) 30~50대 장 건강 고민 여성
주요 성분: 예) 락토바실러스, 비피더스균 5종
```

> 한 번 입력하면 브라우저에 저장됨 (localStorage)
> 다른 탭에서도 자동 불러와짐

## 5단계: 실제 테스트 (1시간)

### 5-1. 키워드 분석

1. 사이드바: 🔍 키워드 분석
2. 텍스트박스에 테스트 키워드 3개 입력:
   ```
   유산균 추천
   장건강 영양제
   프로바이오틱스 효능
   ```
3. "키워드 확장" 클릭 → 관련 키워드 펼쳐짐 → 원하는 것 체크
4. "검색량 분석" 클릭 → 월간 검색량, 경쟁강도 표시
5. "Notion 저장" 클릭 → 키워드 DB에 저장

### 5-2. 채널 배정

1. 사이드바: 📊 키워드 현황
2. "Notion 동기화" 클릭 → 방금 저장한 키워드 표시
3. 각 키워드에서 "배정 채널" 셀 클릭 → 채널 1개 선택:
   - 유산균 추천 → 블로그
   - 장건강 영양제 → 카페SEO
   - 프로바이오틱스 효능 → 지식인

### 5-3. 일괄 생성

1. 사이드바: ⚡ 일괄 생성
2. "Notion에서 키워드 로드" 클릭
3. "3개 키워드 로드됨" 확인
4. 테이블 확인:
   ```
   ☑ 유산균 추천      [블로그]
   ☑ 장건강 영양제     [카페SEO]
   ☑ 프로바이오틱스 효능 [지식인]
   ```
5. 제품 정보 확인 (4단계에서 입력한 것 자동 로드)
6. 작업 계정 선택
7. "일괄 생성 시작" 클릭
8. 프로그레스바 진행 → 완료까지 대기
9. 결과 확인: 3개 모두 ✅ 저장됨
10. Notion 콘텐츠 DB에서 3개 페이지 생성 확인

### 5-4. 쓰레드 테스트

1. 사이드바: 🧵 쓰레드
2. **일상글**: 계정 선택 → 2개 생성 → 결과 확인
3. **물길글**: 셀링 로직 "정보 셔플식" → 키워드 입력 → 생성
4. **게시 테스트**: 결과 카드에서 "즉시 게시" → Threads 확인

### 5-5. 콘텐츠 품질 체크

```
블로그:
  □ 제목에 키워드 포함?
  □ 본문 길이 2000~3000자?
  □ 광고 느낌 안 나는지?
  □ 나만의 키워드만 사용? (제품명 직접 노출 X)

카페SEO:
  □ 카페 분위기에 맞는 톤?
  □ 댓글이 자연스러운지?

지식인:
  □ 질문이 실제 사용자처럼 보이는지?
  □ 답변이 자연스러운지?

쓰레드:
  □ 첫 줄 어그로?
  □ 200자 내외?
  □ 짧은 단문 + 줄바꿈?
  □ SNS 톤 (블로그/카페 톤 아닌지)?
```

> 품질이 마음에 안 들면 프롬프트 수정이 필요합니다.

## 6단계: 서버 안정화 (선택, 권장)

### 6-1. 서버 자동 시작 (Mac 부팅 시 자동 실행)

터미널에서:
```bash
mkdir -p ~/Library/LaunchAgents

cat > ~/Library/LaunchAgents/com.marketing.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.marketing</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/bin/python3</string>
    <string>/Users/iconlms/Desktop/안티그래비티/server.py</string>
  </array>
  <key>WorkingDirectory</key>
  <string>/Users/iconlms/Desktop/안티그래비티</string>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/tmp/marketing.log</string>
  <key>StandardErrorPath</key>
  <string>/tmp/marketing-error.log</string>
</dict>
</plist>
EOF

launchctl load ~/Library/LaunchAgents/com.marketing.plist
```

확인:
```bash
launchctl list | grep marketing
```
`com.marketing`이 보이면 성공.

해제:
```bash
launchctl unload ~/Library/LaunchAgents/com.marketing.plist
```

### 6-2. 데이터 자동 백업 (매일 새벽 2시)

```bash
mkdir -p ~/backup
crontab -e
```

에디터가 열리면 아래 한 줄 추가:
```
0 2 * * * tar -czf ~/backup/marketing-$(date +\%Y\%m\%d).tar.gz -C /Users/iconlms/Desktop/안티그래비티 threads_accounts.json threads_queue.json naver_accounts.json weekly_schedule.json cafe24_token.json performance_history.json 2>/dev/null
```

저장 후 닫기.

확인:
```bash
crontab -l
```

## 7단계: 매일 실무 루틴

### 아침 09:00 — 콘텐츠 생성

1. http://localhost:8000 접속
2. 상단 알림: "콘텐츠 생성 시간입니다 (N개 대기)"
3. ⚡ 일괄 생성 → 키워드 로드 → 체크 → 생성 시작
4. 생성 완료까지 대기 (다른 작업 하면서)

### 오후 14:00 — 콘텐츠 검수

5. 알림: "콘텐츠 검수 시간입니다"
6. Notion 콘텐츠 DB → 생산상태 "초안" 필터
7. 각 콘텐츠 확인 → 수정 → 생산상태 "완성"으로 변경

### 저녁 17:00 — 배포

8. 알림: "콘텐츠 배포 시간입니다"
9. Notion에서 "완성 + 미배포" 필터
10. 각 콘텐츠를 해당 플랫폼에 수동 배포:

**블로그:**
- 네이버 블로그 → 글쓰기
- 제목/본문 복사-붙여넣기
- 발행 → URL 복사

**카페:**
- 해당 카페 → 글쓰기
- 제목/본문 복사-붙여넣기
- 등록 → URL 복사

**지식인:**
- 질문 등록 → URL 복사
- 답변 계정으로 전환 → 답변 등록

11. 대시보드 → 키워드 현황 → 배포URL 입력 + "배포완료"

## 문제 해결

### 서버가 안 켜져요
```bash
lsof -ti:8000 | xargs kill -9  # 포트 점유 프로세스 제거
python3 server.py               # 재시작
```

### Notion 저장이 안 돼요
- .env의 NOTION_TOKEN이 유효한지 확인
- Notion Integration이 해당 DB에 연결되어 있는지 확인
- Notion → DB 오른쪽 상단 ··· → Connections → 해당 Integration 추가

### Threads 게시가 안 돼요
- 토큰 만료 확인: 대시보드 → 쓰레드 → 계정 관리 → "N일 남음" 확인
- 만료됐으면: "OAuth 연결" 다시 클릭
- 일일 한도(250건) 초과 확인

### 일괄 생성에서 키워드가 안 나와요
- 키워드 현황에서 채널이 배정되어 있는지 확인
- 키워드 상태가 "미사용"인지 확인 (이미 사용된 키워드는 안 나옴)

### 스케줄러 알림이 안 와요
- 서버가 켜져 있는지 확인
- 브라우저에서 대시보드가 열려 있어야 알림이 옴 (30초마다 폴링)
