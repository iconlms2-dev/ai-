# 카페SEO 구조 변경 설계 요청서

## 배경

멘토의 카페 외부형 바이럴 마케팅 가이드라인을 분석한 결과, 현재 우리 프로그램의 구조와 상당히 다릅니다.
멘토의 방식을 기준으로 카페SEO 전체 파이프라인을 재설계해야 합니다.

---

## 멘토의 7단계 워크플로우 (기준)

```
1단계. 상위노출 글 정보 수집 (파이썬, 반자동)
  - cafe_scraper.py → 카페탭 1위 글 제목+링크 추출
  - cafe_analyzer.py → 본문, 글자수, 키워드반복횟수, 사진수 분석
  - 분석 결과를 노션 DB에 입력

2단계. MAKE 시나리오 자동 실행 (자동)
  - 95번(노션 읽기) → 89번(Claude 제목 리라이팅) → 88번(Claude 본문) → 92번(Claude 댓글 10개)
  - 93번(노션 제목+댓글 저장) + 94번(노션 본문 저장)
  - 노션 상태: "make 완료"

3단계. 사진 첨부 (수동)
  - 본문 내 [어울릴 사진: 설명] 위치에 실제 이미지 첨부
  - 3~5장, 문단 사이마다 1장씩
  - 노션 상태: "사진첨부 완료"

4단계. 원고 수정 (수동)
  - 제목 수정: 나이대/성별/감정 삽입하여 직관적+자극적으로
  - 본문 수정: 나만의 키워드/링크 삽입, 타사→자사 교체
  - 금칙어 검사: 셀프모아 사이트로 검사 → 위반 단어 대체
  - 댓글 수정: 공감→차선책→해결책→제품→마무리 흐름 검수
  - 노션 상태: "규칙검토완료(사진첨부)"

5단계. 카페에 게시글 등록 (수동)
  - 비행기모드 IP변경 → 시크릿모드 로그인
  - 노션 원고를 메모장 거쳐서 붙여넣기
  - 이미지 삽입, 댓글 허용 체크 후 등록

6단계. 댓글 등록 — "광고의 달인" 프로그램 (반자동)
  - 댓글 10개를 카페 게시글에 자동 등록

7단계. 답글 작성 (반자동)
  - GPT로 각 댓글에 대한 답글 생성
  - 수동 검수 후 카페에 입력
  - 댓글 비활성화
  - 노션 상태: "완료"
```

노션 상태 전이: `(입력)` → `make 완료` → `사진첨부 완료` → `규칙검토완료` → `완료`

## 현재 우리 프로그램 구조

```
대시보드에서 키워드 입력 → API 호출
  → 상위글 크롤링+분석
  → 제목 생성 (처음부터 새로 작성)
  → 본문 생성 (처음부터 새로 작성)
  → 댓글 10개 생성
  → 사진 수집 (샤오홍슈 크롤링+모자이크)
  → AI 검수
  → 노션 저장
한 번에 끝.
```

주요 차이:
- 우리: "처음부터 새 글 작성" / 멘토: "상위글 구조 → 리라이팅"
- 우리: 한 번에 모든 것 자동 처리 / 멘토: AI 초안 → 수동 수정 → 등록 → 댓글 → 답글 단계별 진행
- 우리: 노션 상태 1단계 (저장만) / 멘토: 노션 상태 4단계 전이
- 우리: 답글 생성 없음 / 멘토: 댓글 등록 후 답글까지 생성
- 우리: 금칙어 검사 카페SEO에 없음 (블로그에만 있음)

## 설계 요청 사항

### 목표
멘토의 7단계 워크플로우를 우리 시스템(FastAPI 서버 + 대시보드 HTML)에 반영하되, Make.com은 사용하지 않고 우리 서버가 대신 처리.

### 설계해야 할 것들

#### 1. 전체 파이프라인 재설계
- 멘토 7단계를 우리 시스템에 맞게 재구성
- 각 단계가 API 엔드포인트로 어떻게 분리되는지
- 대시보드 UI에서 사용자가 각 단계를 어떻게 진행하는지
- 노션 상태값 4단계 전이를 어떻게 구현하는지

#### 2. 생성 방식 변경
- 현재: 키워드 → 새 글 작성
- 변경: 상위글 크롤링 → 구조 참고 → 리라이팅 방식으로 생성
- 제목: 상위글 제목을 어미만 바꾸는 리라이팅
- 본문: 경쟁사 글을 참고하되 새로 작성 (멘토 88번 모듈 방식)

#### 3. 수동 작업 중 AI가 대체할 수 있는 부분 분석
멘토의 수동 단계(3~7)에서 AI가 자동으로 처리할 수 있는 것들:
- 사진 첨부: 이미 샤오홍슈 크롤링+모자이크 구현되어 있음
- 금칙어 검사: 셀프모아 API 호출 가능 (블로그에 이미 구현)
- 금칙어 자동 수정: AI에게 대체어 치환 요청 가능
- 제목 수정 (나이대/성별 삽입): AI가 할 수 있음
- 본문 수정 (나만의 키워드/링크 삽입): AI가 할 수 있음
- 댓글 수정: AI가 흐름 검수 가능
- 답글 생성: AI가 할 수 있음
- 댓글 등록: Selenium/API로 가능 (이미 auto-comment 엔드포인트 있음)
- 게시글 등록: Selenium으로 가능하지만 리스크 높음

각 수동 작업에 대해:
- AI 자동화 가능 여부
- 자동화 시 리스크 (계정 정지, 품질 저하 등)
- 사용자 확인이 필요한 지점
- 추천 방식 (완전 자동 / 반자동+확인 / 수동 유지)

#### 4. 대시보드 UI 설계
- 현재: 키워드 입력 → 생성 버튼 → 결과 표시 (1페이지)
- 변경: 단계별 진행 UI
  - 각 단계의 결과를 보여주고
  - 사용자가 확인/수정 후 다음 단계 버튼
  - 또는 "전체 자동"으로 한번에 처리하는 옵션
  - 노션 상태에 따라 작업 대기열 표시

#### 5. 기존 코드 활용 방안
현재 src/api/cafe.py에 이미 있는 것들:
- `_analyze_top_for_cafe()`: 상위글 3개 분석 (사진수/키워드반복/글자수)
- `_search_cafe_top_article()`: 상위 1위 글 URL+제목 검색
- `_crawl_cafe_article()`: 카페 글 크롤링 (제목, 본문)
- `_collect_xhs_for_cafe()`: 샤오홍슈 이미지 수집+모자이크
- `auto-comment` 엔드포인트: 네이버 계정으로 댓글 자동 등록
- `save-notion` 엔드포인트: 노션 저장
- `docx` 엔드포인트: Word 파일 생성

이미 구현된 것들을 최대한 활용하면서 파이프라인 구조를 재설계해야 함.

## 기술 스택
- 백엔드: FastAPI (Python)
- 프론트: dashboard.html (바닐라 JS, 싱글페이지)
- AI: Claude API (본문/댓글), Gemini API (번역/검수)
- DB: Notion API
- 크롤링: requests + BeautifulSoup (네이버), xiaohongshu-cli (사진)
- 자동화: Selenium (댓글 등록)

## 제약사항
- Make.com 사용하지 않음 (우리 서버가 대체)
- 프롬프트 내용은 이 설계에 포함하지 않음 (별도 작업)
- 카페SEO 채널만 먼저 설계 (다른 채널은 나중에)
